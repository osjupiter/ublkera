# known_seeds — カオステストの回帰 seed

過去に何かを捕まえた(またはヒヤリとした)chaos の seed を1ファイル1件で
記録する。`scripts/chaos-vm.sh` はランダムエピソードの**前に**ここの全
seed を必ず再生する。

## 形式

`<短い名前>.env`(シェルで source される):

```sh
# what:   何を捕まえた seed か(1〜2行)
# commit: 発見時の git commit
# date:   2026-07-17
SEED_K=42
OPS_K=300
```

size / granularity / queue 数は seed から決定的に導出されるので、
SEED_K と OPS_K だけで同じ操作列が再生される。

## 運用の原則

- **恒久的な回帰は専用テストに固定する**のが主で、seed の保存は従。
  実カーネルを挟むため seed の再現は確率的であり(SST の準決定性)、
  「同じ seed を回せば必ず踏む」保証はない。バグの原因を特定したら、
  ユニットテスト(era.rs)/ シナリオ(scenarios/)/ e2e の節として
  決定的に踏むテストを書き、seed は当時の再現手段+統計的な網として残す
- 修正済みかどうかに関わらず消さない(将来のリグレッションに対する網)
- ファイル名は捕まえた内容がわかる短い名前に(例: `lost-4k-write.env`)
